# תוכנית בדיקות — Influ Capital

המסמך הזה מגדיר **איך אנחנו מוודאים שהאתר עובד מושלם תמיד** — מהיר, לא נשבר, נראה טוב, נגיש, בכל מסך וכל תנאי רשת.

הרעיון המרכזי: **לולאות סגורות על KPI ברורים.** מגדירים יעד מדיד → מריצים סשן בדיקה → מודדים → אם מתחת לסף פותחים משימה ב‑`TASKS.md`, מתקנים, ומריצים שוב עד ירוק.

---

## שכבות הבדיקה (תמיד, ללא פשרה)

1. **Unit tests** — כל פיסת לוגיקה שניתן לבודד (בניית קישורי טלפון/וואטסאפ, ניתוב, ולידציית טופס) מקבלת בדיקת יחידה. ראה `tests/unit/`.
2. **בדיקות מקצה‑לקצה (E2E)** — Playwright שמדמה משתמש אמיתי על כל המסלולים והמכשירים. ראה `tests/e2e/` ו‑[`playwright-runbook.md`](./playwright-runbook.md).
3. **סשני AI אקספלורטיביים** — אחת לתקופה Claude מריץ Playwright בצורה חוקרת (לא רק תרחישים קבועים): נכנס לאתר, גולש כמו משתמש אמיתי לאור ה‑[user stories](./user-stories.md), ובודק שהכול מתפקד, מהיר ויפה — במגוון מסכים, מהירויות רשת ומסלולי גלישה.
4. **Lighthouse** — כל עמוד/מסלול נבדק מול ספי ביצועים, נגישות, SEO ו‑Best Practices. ראה [`lighthouse-runbook.md`](./lighthouse-runbook.md).

---

## תהליך הלולאה הסגורה

```
KPI ברורים  ──►  סשן בדיקה (E2E + AI + Lighthouse)  ──►  מדידה מול ספים
     ▲                                                          │
     │                                                          ▼
   re-run  ◄──  תיקון (משימה ב-TASKS.md)  ◄──  כשל / מתחת לסף?  ──► עבר? → לוג ירוק
```

- היעדים המדידים מוגדרים ב‑[`kpi.md`](./kpi.md).
- כל סשן מתועד תחת `results/` לפי התבנית `results/_template.md`.
- כל ממצא שדורש תיקון נפתח כמשימה ב‑`TASKS.md` שבשורש הפרויקט.

## מתי מריצים

- **לפני כל עלייה לאוויר / שינוי משמעותי.**
- **אחת לתקופה** כסשן רוחבי יזום (Claude ישאל אם להריץ — ראה CLAUDE.md).
- **בכל פעם שה‑user stories מתעדכנים** — סטוריז חדשים = תרחישים חדשים לבדוק.

## איך מריצים (מהיר)

```bash
cd בדיקות
npm install            # פעם ראשונה בלבד
npx playwright install # התקנת דפדפנים, פעם ראשונה
npm run test:e2e       # בדיקות מקצה לקצה בכל המכשירים
npm run test:unit      # בדיקות יחידה
npm run lighthouse     # ראה lighthouse-runbook.md לפירוט per-page
```

> האתר הוא קובץ סטטי יחיד (`../index.html`) עם ניתוב hash. ה‑Playwright config מרים שרת סטטי מקומי אוטומטית.
