# Playwright Runbook — סשן בדיקה

איך מריצים סשן Playwright — גם תרחישים קבועים (`tests/e2e/`) וגם **סשן AI חוקר**.

---

## הרצה של התרחישים הקבועים

```bash
cd בדיקות
npm install && npx playwright install   # פעם ראשונה
npm run test:e2e                        # כל המכשירים
npm run test:e2e -- --project=mobile    # מכשיר ספציפי
npm run report                          # פתיחת דוח HTML
```

ה‑config (`playwright.config.js`) מרים שרת סטטי מקומי על `../index.html` ומריץ על הפרויקטים: `desktop`, `mobile-iphone`, `mobile-pixel`, `tablet`.

---

## סשן AI חוקר (אחת לתקופה)

המטרה: לא רק לרוץ על תרחישים שכבר כתבנו, אלא **להשתמש באתר כמו משתמש אמיתי** ולחפש מה נשבר/מכוער/איטי.

### צעדים ל‑Claude כשמאשרים סשן

1. קרא את `user-stories.md` (המקור לתרחישים) ואת `kpi.md` (הספים).
2. הרם את האתר מקומית והפעל Playwright (MCP אם זמין, אחרת `tests/e2e`).
3. עבור על **מטריצת המכשירים × הרשת** מתוך `kpi.md`. לכל שילוב:
   - גלוש בכל המסלולים (`#/`, `#/ליווי-אישי`, `#/קורס`, `#/הרצאות`, `#/אודות`, `#/צור-קשר`).
   - הפעל כל CTA (וואטסאפ / חיוג / מייל / טופס), אקורדיון, וידאו, פאנל נגישות, פופאפ יציאה, תפריט מובייל.
   - צלם screenshot בכל עמוד ובדוק ויזואלית: חיתוכי טקסט, RTL, אייקוני Lucide, תמונות, layout shift.
   - אסוף שגיאות console ובקשות רשת שנכשלו.
4. מדוד מול הספים. כל חריגה → רשום ב‑`results/` ופתח משימה ב‑`TASKS.md`.
5. סכם תוצאה: ירוק / אדום + רשימת ממצאים.

### מה בודקים בכל מסך (checklist מהיר)

- [ ] כל הטקסט קריא, ללא גלישה/חיתוך, RTL תקין
- [ ] אייקונים (Lucide) ותמונות נטענים ובגודל נכון
- [ ] כל כפתור/קישור עובד ומוביל ליעד הנכון
- [ ] אין שגיאות console / בקשות 404
- [ ] אין layout shift בטעינה או באינטראקציה
- [ ] אינטראקציות מגיבות מהר (< 200ms)
- [ ] ניווט מקלדת + focus ויזואלי תקינים
- [ ] נראה טוב ב‑3 מהירויות הרשת
