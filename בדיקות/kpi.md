# KPI וספים מחייבים

היעדים המדידים של הלולאה הסגורה. **כל מסלול חייב לעבור את כל הספים** כדי שהסשן ייחשב ירוק. סף שנכשל = משימת תיקון ב‑`TASKS.md`.

---

## Lighthouse (נבדק לכל עמוד, מובייל + דסקטופ)

| קטגוריה | סף מינימום |
|---------|-----------|
| Performance | ≥ 90 |
| Accessibility | ≥ 95 |
| Best Practices | ≥ 95 |
| SEO | = 100 |

> נגישות ו‑SEO הם עקרונות יסוד של הפרויקט (ראה CLAUDE.md) — לכן הסף שלהם גבוה במכוון.

## Core Web Vitals (מובייל, throttle ברירת מחדל)

| מדד | יעד |
|-----|-----|
| LCP (Largest Contentful Paint) | < 2.5s |
| CLS (Cumulative Layout Shift) | < 0.1 |
| INP / TBT | INP < 200ms · TBT < 200ms |
| FCP | < 1.8s |

## תקציב משקל עמוד (Performance Budget)

| נכס | תקרה |
|-----|------|
| HTML | < 90 KB |
| סה"כ JS (כולל CDN) | < 150 KB |
| סה"כ משקל עמוד ראשוני | < 1 MB |
| בקשות רשת | < 25 |

## תקינות פונקציונלית (E2E)

- 100% מסיפורי המשתמש ב‑`user-stories.md` עוברים.
- **אפס** שגיאות/אזהרות ב‑console בכל מסלול.
- **אפס** קישורים שבורים (כולל `tel:` / `wa.me` / `mailto:`).
- כל הדפים נטענים תחת 3 שניות ב‑Fast 3G.

## מטריצת מכשירים (חייב לעבור בכולם)

iPhone SE · iPhone 14 · Pixel 5 · iPad · Desktop 1440px+

## מטריצת רשת

Online (ללא הגבלה) · Fast 3G · Slow 4G

## הגדרת "ירוק"

סשן ירוק = **כל** הספים לעיל עוברים על **כל** המכשירים והמסלולים, וכל סיפורי המשתמש עוברים. אחרת — אדום, ופותחים משימות תיקון עד שחוזרים לירוק.
