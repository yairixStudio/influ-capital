# BLUEPRINT.md — עץ האתר (עמודים · סקשנים · טמפלטים)

מקור אמת יחיד למבנה האתר: אילו **עמודים** קיימים, אילו **סקשנים** בכל עמוד, באיזה **סדר**, מה **מטרת** כל סקשן,
ואילו סקשנים חולקים **טמפלט** (רכיב חוזר). מאושר על ידי המשתמש.

**איך משתמשים:** לפני בנייה/שינוי של עמוד או סקשן — לעבוד לפי המבנה כאן. סקשן/עמוד חדש שהמשתמש מאשר →
לעדכן קודם את המסמך הזה, ורק אז לבנות. הקופי עצמו נשאב מ‑`versions/COPY.md`, פרטי הקשר מ‑`contact.json`,
והעיצוב/טוקנים מ‑`brand.css` — המסמך הזה מגדיר **מבנה**, לא ניסוחים או עיצוב.

**כלי עריכה:** העץ נערך ויזואלית ב‑`tools/blueprint.html` (כלי תכנון פנימי, RTL). שם אפשר לגרור/לשכפל/לערוך,
לצבוע סקשנים זהים לפי טמפלט, ולייצא `JSON`/`MD`. כדי לסנכרן עדכון: לערוך בכלי → לייצא JSON → לבקש מ‑Claude לעדכן את המסמך הזה.

**מוסכמות:** `· מטרה:` מתארת את הסקשן שמתחתיה · `[טמפלט]` = רכיב חוזר שמשרת כמה סקשנים (אותו צבע בכלי).
**הירו בכל עמוד.** שלושת עמודי השירות זהים בסדר סוגי הסקשנים: `hero‑נחיתה → כאב → תהליך → בידול → המלצות → cta`.
